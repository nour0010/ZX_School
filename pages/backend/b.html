دليل شامل لمسار تعلم الـ back-End من الصفر إلى الاحتراف

ما هو الـ Backend؟
الـ Backend في برمجة المواقع هو الجزء المسؤول عن معالجة البيانات، تشغيل العمليات الخلفية، وإدارة قواعد البيانات. هو ما يحدث "وراء الكواليس" عند تفاعل المستخدم مع الموقع ويعتمد علئ:

1-node.Js

ببساطة، Node.js يحول JavaScript من لغة تعمل فقط في المتصفح إلى لغة يمكن استخدامها في تطوير الخوادم والتطبيقات المستقلة.

أهميتها:
1. إنشاء خوادم ويب سريعة وفعالة باستخدام Express.js.
2. معالجة الطلبات المتعددة بكفاءة بسبب النموذج غير المتزامن.
3. دعم تطبيقات الزمن الحقيقي مثل الدردشة والإشعارات.
4. تكامل قوي مع قواعد البيانات مثل MongoDB وMySQL.
5. سهولة بناء API وتطوير الواجهات الخلفية.
6. توحد لغة البرمجة (JavaScript) بين الواجهة الأمامية والخلفية.
7. أداء عالي مع تطبيقات SPA مثل React وVue.js.

أُطر العمل (Frameworks) في node.Js

أُطر العمل في Node.js هي مكتبات وأدوات برمجية تُستخدم لتسهيل تطوير تطبيقات الويب والخوادم باستخدام Node.js، حيث توفر بنية جاهزة وتنظم الكود، مما يُسرّع عملية التطوير ويجعلها أكثر كفاءة.

اليك اشهرها:
Express.js – أشهر إطار عمل خفيف لإنشاء خوادم ويب وواجهات API بسهولة.

2-قاعده البيانات MySQL

MySQL هي قاعدة بيانات تخزّن البيانات في جداول وترتّبها بشكل منظم، مثل جدول أسماء الطلاب مع درجاتهم. تُستخدم في المواقع والتطبيقات لحفظ البيانات واسترجاعها بسرعة، مثل تخزين معلومات المستخدمين في فيسبوك أو المتاجر الإلكترونية.

لماذا MySQL؟
سريعة وسهلة الاستخدام.
مجانية ومفتوحة المصدر.
تعمل مع لغات برمجة كثيرة مثل PHP وNode.js وPython.
مناسبة للمواقع والتطبيقات الكبيرة والصغيرة.
ببساطة، MySQL تساعدك في تخزين البيانات والوصول إليها بسهولة وكفاءة!

أطر عمل MySQL مع Node.js

Sequelize – إطار عمل ORM يسهل التعامل مع MySQL باستخدام JavaScript.

لماذا Sequelize؟
يسهل التعامل مع MySQL باستخدام JavaScript بدلًا من SQL.
يدعم العلاقات بين الجداول مثل المستخدمين والطلبات.
يقلل الأخطاء ويحمي من SQL Injection.
يدعم الترحيلات (Migrations) لتحديث قاعدة البيانات بسهولة.
يعمل مع قواعد بيانات متعددة مثل PostgreSQL وSQLite بجانب MySQL.
